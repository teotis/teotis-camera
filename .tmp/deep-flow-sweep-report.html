发现的问题

P2 git pack 目录存在 macOS 资源分叉文件

证据: .git/objects/pack/._pack-* (20 个文件) 导致每次 git 操作输出 non-monotonic index 错误。

  • 影响: 掩盖真实 git 状态，初始 status 误报 clean
  • 文件: .git/objects/pack/._pack-*
P2 源码树存在 ~30 个 macOS 元数据文件

证据: find -name "._*" 找到散落在 app/, docs/, res/xml/ 等目录的 ._* 文件。

  • .gitignore 已有 ._* 规则，但磁盘上仍残留
  • 如 ._SessionPreviewRenderModelTest.kt 看起来像测试文件
Info CameraX 适配器 3437 行巨型文件

CameraXCaptureAdapter.kt 包含绑定、拍照、录像、变焦、亮度、旋转、Live Photo、多帧等全部设备适配逻辑。功能正确且有测试覆盖，但可维护性成本高。

Info 真机验证缺口

Live Photo / Motion Photo 的 MediaCodec 编码质量、设备兼容性、快门声音时机和触感反馈仍需真机验证。

已应用修复

1. .git/objects/pack/._pack-* — 删除 20 个 macOS 资源分叉文件，消除 non-monotonic index 错误

2. 源码树 ._* — 批量删除约 30 个散落在源码目录的 macOS 元数据文件
