4. 主导约束判断

主导约束: 仓库体积膨胀（78GB）与 app 模块扁平化耦合的叠加效应。

为什么优先于其他可见重构: SessionUiRenderModel 2325 行虽然更"吸引眼球"，但它是能力阻塞型债务的症状而非根因。根因是: (1) app 模块 40+ 文件无分包，职责边界模糊导致任何 UI 变更都需理解全局；(2) 仓库 78GB 使得 git 操作、CI clone、worktree 管理全部变慢，每次开发迭代都承受隐性税。先瘦身 + 分包，后续拆解 SessionUiRenderModel 的成本会大幅下降。

受影响能力: 构建速度、增量效率、新开发者上手速度、模块独立交付能力。

不作为的机会成本: 每次 git 操作多等数秒；每次新增功能需理解 40+ 文件的隐式职责；worktree 管理持续消耗磁盘。

置信度: 高（磁盘数据可直接验证）。可证伪条件: 如果 worktree 中存在大量未合并的关键工作，则清理需更谨慎。
