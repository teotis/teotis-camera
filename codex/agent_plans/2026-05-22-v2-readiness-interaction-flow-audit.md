# 并行任务 B：用户交互流畅性审查（非多模态）

## 1. 目标

验证关键交互路径是否“可达、可执行、可恢复”，并识别阻塞点。

## 2. 核心路径

1. 冷启动到首帧
2. 模式切换（Photo/Video/Pro 等）
3. 快门拍照与连拍/快速重复拍摄
4. 录像开始/停止
5. 面板开合与参数调整
6. 权限拒绝后恢复

## 3. 审查方法

- 结合 coordinator/session 相关测试
- 抽取交互时序（事件 -> 状态 -> 副作用）
- 记录每条路径的“中断点与恢复点”

## 4. 产出

- `Interaction-Flow-Audit.md`
- `Flow Risk Table`（步骤、预期、实际、风险、归属模块）
- 每条路径至少 1 个可执行回归建议

## 5. 边界

- 不做 UI 美学评价
- 不改动架构边界
