# 并行任务 A：UI 设计逻辑自洽静态审查（非多模态）

## 1. 目标

审查 UI 设计逻辑是否在代码层可映射、可维护、无明显自相矛盾。

## 2. 输入

- `codex/v2_ui/*.md`
- `app/src/main/java/com/opencamera/app/*`（含 camera UI 相关）
- `SessionUiRenderModel*` 相关实现与测试

## 3. 审查清单

1. 信息架构：主入口/次入口分层是否稳定
2. 状态语义：同一状态是否存在多套命名/多套显示逻辑
3. 组件职责：UI 是否仅 render+dispatch，无 runtime owner 泄漏
4. 模式一致性：不同模式下共用组件行为是否一致
5. 文案与 i18n：关键状态文案是否可本地化且不歧义

## 4. 产出

- `UI-Static-Audit.md`
- 问题表（字段：位置、现象、风险等级、建议）
- 至少 10 个高价值发现（若不足需说明为何）

## 5. 验证建议

- 以单测和 render model 断言为主
- 不做视觉主观结论（交由多模态主控）
