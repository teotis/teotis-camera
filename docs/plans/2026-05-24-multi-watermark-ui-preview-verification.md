## UI Product Rules

Selector copy must communicate the output behavior, not implementation detail:

```text
Pure Text
Clean text over the image | model, time, params | position Bottom Left

Blur Four Border
Blurred image frame | model, time, location, params | background Light Blur
```

Detail controls:

- Pure text: placement, text size, opacity.
- Classic overlay: placement, text size, opacity.
- Travel polaroid: placement, text size, opacity, frame background.
- Retro frame: placement, text size, opacity, frame background.
- Blur four border: bottom placement, text size, opacity, blur background.

Do not display a disabled frame background row for pure text. Hiding irrelevant controls is clearer than showing unsupported controls inside the same template.

## Preview Hint Rules

Preview is a promise indicator, not final output rendering.

For `pure-text`:

- Draw the watermark hint text at the selected placement.
- Do not draw a backing rectangle.
- Use the same opacity scaling already present in `PreviewEffectAdapter`.

For `blur-four-border`:

- Draw a thin four-side frame hint around the active preview content bounds.
- Draw the preview text in the lower side of that hint.
- Do not attempt real blur in preview; real blur belongs to saved output rendering.
- Keep the hint subtle so it does not compete with focus reticle, countdown, or composition grid.

## Visual QA Owned By Multimodal Agent

After text-only implementation passes, a multimodal agent should inspect real saved outputs:

- Capture or generate one saved JPEG for `pure-text`.
- Capture or generate one saved JPEG for `blur-four-border`.
- Confirm pure text has no card/backing rectangle and remains legible over dark and bright image regions.
- Confirm blurred four-border shows blur on all four sides, keeps the center photo unblurred, and keeps the lower text from crowding the image.
- Confirm thumbnail/gallery opens the saved, postprocessed JPEG rather than a raw preview snapshot.
- Confirm settings labels do not clip on narrow screens.

This visual QA should not block text-only implementation, but final product acceptance should not claim visual polish until these checks are done.
