## 分析维度

### 1. 界面元素识别
- **按钮**：位置、大小、形状、颜色、文字、图标
- **滑块**：位置、大小、颜色、刻度、标签
- **菜单**：位置、大小、层级、选项、分隔符
- **面板**：位置、大小、背景、边框、阴影
- **输入框**：位置、大小、边框、提示文字、状态
- **图标**：位置、大小、颜色、含义、一致性

### 2. 布局结构分析
- **对齐方式**：左对齐、右对齐、居中、分散对齐
- **间距系统**：元素间距、边距、内边距
- **层次结构**：视觉层次、信息层次、功能层次
- **网格系统**：列数、行数、栅格对齐
- **响应式布局**：屏幕适配、横竖屏切换

### 3. 组件设计分析
- **按钮组件**：主要按钮、次要按钮、文字按钮、图标按钮
- **滑块组件**：亮度滑块、曝光滑块、变焦滑块
- **菜单组件**：下拉菜单、侧边菜单、底部菜单、弹出菜单
- **面板组件**：设置面板、日志面板、预览面板
- **卡片组件**：设置卡片、模板卡片、预览卡片

### 4. 交互组件分析
- **点击区域**：按钮大小、触摸目标、点击反馈
- **滑动区域**：滑块范围、滑动方向、滑动反馈
- **拖拽区域**：拖拽目标、拖拽反馈、放置区域
- **长按区域**：长按触发、长按反馈、长按菜单

## 分析方法

### 1. 逐张分析
对每张截图进行独立分析，识别所有界面元素和布局结构。

### 2. 对比分析
对比不同截图，识别组件设计的一致性和差异性。

### 3. 尺寸估算
估算界面元素的尺寸、间距、边距等参数。

### 4. 颜色提取
提取界面元素的颜色值，分析颜色系统。

## 产出要求

### 报告结构
1. **摘要**：总体发现和关键问题
2. **逐张分析**：每张截图的详细分析
3. **组件清单**：所有识别的组件及其属性
4. **布局分析**：布局结构、对齐方式、间距系统
5. **优化建议**：具体的优化点和实施方向

### 优化建议维度
- **布局优化**：间距调整、对齐改进、层次优化
- **组件优化**：按钮设计、滑块改进、菜单优化
- **交互优化**：点击区域、反馈机制、操作流程
- **视觉优化**：颜色调整、字体改进、图标优化
