# Status Files

Each package executor writes to its own status file. Do NOT edit `INDEX.md`.

## How To Use

1. Fill in your assigned `<package-id>.md`.
2. Include command evidence, not just a summary claim.
3. Do not edit another package status file.
4. Leave unresolved risks visible for the final auditor.
