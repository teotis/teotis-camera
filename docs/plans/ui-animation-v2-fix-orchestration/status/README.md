# Status Files

Each package executor writes only to its own status file. Do not edit `INDEX.md` from an implementation package.

## How To Use

1. Open the assigned `status/<package-id>.md` file.
2. Fill in each section as work proceeds.
3. Record exact verification commands and concise output summaries.
4. Do not edit another package's status file.

The final integration auditor reads all status files and writes the final conclusion.
